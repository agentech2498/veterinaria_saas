# Instrucciones para IA

Antes de modificar código:

Realizar auditoría.

No asumir funcionamiento.

No romper funcionalidades.

No modificar diseño.

No modificar UX.

No crear deuda técnica.

No agregar funcionalidades no solicitadas.

No realizar cambios masivos.

Trabajar por fases.

Esperar aprobación.

Cada cambio debe incluir:

Problema

Causa raíz

Beneficio

Archivos modificados

Riesgo

Validación

Pruebas
