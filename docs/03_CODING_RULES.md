# Reglas del Proyecto

Nunca romper funcionalidades existentes.

Nunca realizar parches.

Nunca ocultar errores.

No usar hacks.

No introducir deuda técnica.

No duplicar código.

No modificar diseño sin autorización.

No cambiar endpoints existentes salvo justificación.

No cambiar modelos de datos salvo aprobación.

Siempre justificar cambios.

Siempre realizar auditoría antes de modificar.

Siempre validar después de modificar.

Compilar antes de finalizar.

Backend debe seguir siendo la única fuente de verdad.
